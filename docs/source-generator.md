# Source generator (XSpecification.SourceGenerator)

`XSpecification.SourceGenerator` is a Roslyn 4.x incremental generator that:

1. Discovers every class deriving from `XSpecification.Core.SpecificationBase<TModel, TFilter, TResult>`.
2. Generates a static accessor table `IReadOnlyDictionary<string, Func<TFilter, object?>>` for the
   filter type — replaces `PropertyInfo.GetValue` on the hot path.
3. Reports `XSPEC001` (warning) for filter properties that are not mapped to a model property and
   are not registered via `HandleField` / `IgnoreField`.

## Adding it

```bash
dotnet add package XSpecification.SourceGenerator
```

The package is a development dependency (`<DevelopmentDependency>true</DevelopmentDependency>`),
so consumers don't ship the analyzer DLL with their app — only the generated code.

## Generated code

For a specification:

```csharp
namespace MyApp.Search;

public sealed class CustomerSpec : SpecificationBase<Customer, CustomerFilter>
{
    public CustomerSpec(...) : base(...) { }
}
```

the generator emits:

```csharp
// <auto-generated/>
namespace MyApp.Search;

internal static class CustomerSpecAccessors
{
    public static readonly IReadOnlyDictionary<string, Func<MyApp.Search.CustomerFilter, object?>> FilterPropertyAccessors =
        new Dictionary<string, Func<MyApp.Search.CustomerFilter, object?>>
        {
            ["Name"] = static f => f.Name,
            ["StatusIn"] = static f => f.StatusIn,
            // ... one entry per filter property
        };
}
```

You can use this dictionary directly to side-step the reflection-based fallback in
`SpecificationBase.CreateResultFromFilterProperty`.

## Diagnostics

| Id        | Severity | Description                                                  |
|-----------|----------|--------------------------------------------------------------|
| `XSPEC001` | Warning  | Filter property is not mapped to a model property            |
| `XSPEC100` | Hidden   | Specification class detected (informational)                 |

Suppress via `#pragma warning disable XSPEC001` or by adding the property to your spec's
constructor with `IgnoreField`.
